# Журнал изменений

Здесь фиксируются заметные изменения сервиса для учеников и администратора.
Проект использует семантические версии `MAJOR.MINOR.PATCH`:

- `MAJOR` — несовместимые изменения или крупная перестройка продукта;
- `MINOR` — новая функциональность без поломки существующих сценариев;
- `PATCH` — исправления, тексты и небольшие улучшения.

## Правило публикации

Перед каждым push с изменениями продукта на GitHub необходимо:

1. Просмотреть изменения после последней версии.
2. Выбрать новый номер по правилам выше — как минимум увеличить `PATCH`.
3. Обновить версию в `service/package.json` и `service/package-lock.json`.
4. Добавить в этот файл дату и понятное человеку описание изменений.
5. Выполнить `npm run lint` и `npm run build` в папке `service/`.
6. Только после этого создавать коммит и выполнять push.

## [1.1.0] — 2026-07-17

Первый функциональный релиз после первоначального развёртывания на Beget.

### Добавлено

- Полноценные уроки курса и общий импорт контент-пакетов уроков 1–8.
- Управление прогрессом учеников в админ-панели, включая безопасный сброс отдельных результатов.
- Управление аккаунтами: создание, редактирование, блокировка, завершение активных сессий и удаление.
- Учёт входов, активности по дням, времени в уроках и повторного изучения материалов.
- Пояснения к правильным и неправильным ответам теста со ссылками на соответствующие блоки урока.
- Частичное начисление баллов в вопросах с несколькими правильными вариантами.
- Кнопка повторного прохождения теста с очисткой новой попытки.

### Изменено

- Дополнительные уроки сделаны компактными и открываются последовательно по мере прохождения курса.
- Варианты ответов и задания на сортировку перемешиваются для новой попытки, сохраняя стабильный порядок внутри текущей попытки.
- Результаты теста получили подробный разбор, сворачиваемые пояснения и более понятное отображение баллов.
- Административные разделы прогресса, уроков и аккаунтов собраны в более понятную структуру.
- Авторизация учитывает состояние аккаунта и позволяет принудительно завершать старые сессии.

### Исправлено

- Карточки достижений на странице прогресса отображаются корректно.
- Прогресс уроков корректно пересчитывается после импорта и административных действий.
- При попытке завершить заполненный не полностью тест подсвечиваются только пропущенные вопросы.
- Неполная попытка не сохраняется и не начисляет баллы.
- Уже введённые ответы, текст и частично заполненная сортировка больше не сбрасываются после предупреждения о пропусках.
